Reimport All Calendars

Deletes all existing calendar entries for every organization that has a calendar URL configured, then fetches and re-classifies everything from scratch. Use this to fix stale data — for example, entries that show "Shkiya:" instead of "Plag:" because they were imported before the classifier was improved.

  • Applies to all orgs with a calendar URL (ignores useScrapedCalendar flag)
  • Re-runs the full classification pipeline on every event
  • Fixes notes/Plag annotations on reimport
  • After reimport, rematerialize to update live schedule

Rematerialize All

Triggers a full rebuild of the materialized calendar_events table from Minyan rules and imported calendar entries. This is the same job that runs automatically every Sunday at 2 AM. Run this after reimporting calendars or editing Minyan schedules to push changes live immediately.

  • Deletes and rebuilds all RULES-sourced events
  • Preserves IMPORTED and MANUAL events
  • Covers the rolling 11-week window
Reimport All Calendars?

This will delete all existing calendar entries for every organization that has a calendar URL, then re-fetch and re-classify from scratch.

This operation may take a minute or two. The page will reload when it completes. After reimporting, run Rematerialize All to push the updated data to the live schedule.

Rematerialize All?

This will trigger a full rebuild of the materialized calendar events table. All RULES-sourced events will be deleted and regenerated. IMPORTED and MANUAL events are preserved.

This is safe to run at any time and typically takes a few seconds.
